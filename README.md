# お散歩ビンゴ

散歩中に見つけたものをマークしてビンゴを目指すゲームです。

🌐 **プレイはこちら** → https://b-w-hiroki.github.io/osanpo-bingo/game.html

---

## できること

### 🙂 ひとりで遊ぶ

散歩に出かけて、5×5のビンゴボードのお題を探して遊ぶ基本モードです。

**遊び方の選択肢**
- **📷 写真で記録**：お題を見つけたらマスをタップ → 写真を撮ってマーク
- **✓ マークだけ**：タップするだけでマーク（写真なしで手軽に）

**ボードのカスタマイズ**
- **難易度3段階**：かんたん（よく見るもの） / ふつう（少し探す） / むずかしい（レアなもの）
- **フリー入力マス**：0〜12マス分、自分だけのオリジナルお題を設定できる（最大20文字）
- **作り直す**：ゲーム中でもお題をシャッフルして新しいボードに切り替えられる

---

### 👥 みんなで遊ぶ（通常グループモード）

同じ合言葉を共有してグループプレイができます。

**グループプレイの特徴**
- 合言葉を決めてゲームを作り、友達に共有（最大5文字・自動生成ボタンあり）
- 同じ合言葉 + 同じ難易度なら**同じお題プール**から出題される
- 各自のボードレイアウトはシャッフルで異なる（自分だけのシートで遊べる）
- 参加人数を設定できる

**遊び方のイメージ**：家族や友達でそれぞれスマホを持って散歩。同じお題を探しながら、先にビンゴを揃えた人が勝ち。

---

### ⚔️ バトルモード（陣取りビンゴ）

同じ合言葉で遊ぶプレイヤーが、マスを先着で取り合う競争モードです。

**バトルの仕組み**
- お題を見つけて**写真を撮ってアップロードした人がそのマスを取得**（先着1名限定）
- 他のプレイヤーが先に取ったマスはグレーになり取れない
- 自分が取ったマスは緑で表示
- 4秒ごとに全員の取得状況をリアルタイムで同期

**画面で見えるもの**
- ✓ マーク：自分が取ったマス数
- 🛡️ 相手取得：他のプレイヤーが取ったマス数

**遊び方のイメージ**：散歩しながら同じお題を奪い合う。先に写真を撮った人がそのマスをゲット。多くのマスを取りビンゴを揃えた人が勝ち。

---

### 📷 写真機能

- カメラで直接撮影、またはライブラリから選択
- アップロード前にプレビューして確認・撮り直しできる
- 写真は自動圧縮（最大800px）でストレージを節約
- 各マスに1枚保存、削除も可能
- 写真付きマスはボード上にサムネイル表示

---

### 🏁 結果・共有機能

ゲーム終了後にまとめ画面が表示されます。

- **結果カード**：日付・プレイ時間・ビンゴ数・マーク数を表示
- **グループ名入力**：「家族で散歩」「友達3人で」など記録できる（最大50文字）
- **画像保存**：結果画面をそのまま画像としてダウンロード
- **SNS共有**：テキストで結果をシェア（対応ブラウザではWebShare API使用）

---

### ⚙️ その他の機能

- **オフライン対応（PWA）**：一度開けばインターネットなしでも遊べる
- **自動保存**：LocalStorageで状態を保持。ブラウザを閉じても続きから再開できる
- **合言葉のコピー**：画面下の合言葉をタップするとクリップボードにコピー
- **キーボード操作対応**：Enter でマーク、P で写真、Tab で移動
- **エクスポート／インポート**：JSON形式でデータのバックアップ・復元が可能

## バトルモードのセットアップ

バトルモードは**設定なしでも遊べます**（同一端末内での動作）。  
端末間でリアルタイムに競い合いたい場合は Supabase の設定が必要です。

1. `battle-config.example.js` を `battle-config.js` にコピー
2. `battle-config.js` に Supabase の URL と anon key を入力
3. Supabase に `battle_cell_owners` テーブルを作成

```
テーブル: battle_cell_owners
カラム: room_code, topic_key, cell_index, owner_user_id
UNIQUE制約: (room_code, topic_key)
```

`battle-config.js` は `.gitignore` に登録されているため、誤って公開される心配はありません。

## 開発者向け

### ローカルで起動する

```bash
npx --yes serve -l 3000
```

ブラウザで `http://localhost:3000/game.html` を開く。

### ファイル構成

```
game.html           … ゲーム画面
app.js              … ゲームロジック（OsanpoBingo クラス）
styles.css          … スタイル
topics.js           … お題一覧・難易度別選択ロジック
battle-config.js    … Supabase接続設定（gitignore対象）
battle-config.example.js … 設定ファイルのテンプレート
app-battle.js       … バトルモード関連メソッド抜粋（レビュー参照用）
```

### 技術スタック

- HTML5 / CSS3 / JavaScript (ES6+)
- PWA（Service Worker・Web App Manifest）
- Supabase REST API（バトルモードのみ・任意）

### ブラウザ対応

Chrome / Firefox / Safari / Edge
