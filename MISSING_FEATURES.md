# お散歩ビンゴ - Web版との機能比較と不足機能

## 📊 Web版の主要機能（公式情報から）

Web版お散歩ビンゴ（solomaker.dev）の説明：
> 散歩をしながらお題の写真を撮ってビンゴを目指そう！

**明示されている機能**:
1. ✅ お題の写真を撮ってビンゴを目指す
2. ⚠️ **溜まっていくポイントで自分のお散歩実績を実感**
3. ✅ 自分でお題を追加できる
4. ✅ エクスポート／インポート機能つき
5. ✅ PWAだからアプリとして使える
6. ✅ ビンゴ画面は画像としてシェア可能

---

## 🚨 現在の仕様に不足している機能

### 1. ポイントシステム（重要！）⭐⭐⭐
**Web版**: 「溜まっていくポイントで自分のお散歩実績を実感」

**現在の仕様**:
- 統計表示（マーク数、写真数、ビンゴ数）はある
- しかし**ポイント**という概念がない
- **累計・履歴**の概念がない

**不足している要素**:
```javascript
// 必要なデータ
{
  totalPoints: 0,           // 累計ポイント
  totalGames: 0,            // プレイ回数
  totalBingos: 0,           // 累計ビンゴ数
  totalPhotos: 0,           // 累計写真数
  achievements: [],         // 達成したお題の履歴
  bingoHistory: []          // ビンゴ達成の履歴
}
```

**ポイントの計算方法（推測）**:
- マーク1つ = 1ポイント
- 写真付きマーク = 2ポイント
- ビンゴ1本 = 10ポイント
- 全マス達成 = ボーナス50ポイント

**必要なUI**:
- 累計ポイント表示
- 実績/達成履歴画面
- レベルやバッジ（オプション）

---

### 2. 達成履歴・実績管理（重要！）⭐⭐⭐
**Web版**: 「お散歩実績を実感」= 過去のデータを見られる

**現在の仕様**:
- **履歴の保存がない**
- 現在のゲームのデータしか残らない
- 過去にどんなお題を達成したか分からない

**不足している要素**:
```javascript
// 達成履歴
{
  bingoHistory: [
    {
      date: '2026-02-10',
      bingoCount: 2,
      totalMarks: 18,
      totalPhotos: 12,
      pointsEarned: 50,
      completedTopics: ['信号', 'ネコ', '犬', ...]
    },
    // ...
  ]
}
```

**必要なUI**:
- 履歴画面（カレンダー形式？）
- 累計統計（総ビンゴ数、総お題達成数など）
- 達成率グラフ（オプション）

---

### 3. 写真が必須かオプションか？⭐⭐
**Web版**: 「お題の**写真を撮って**ビンゴを目指す」

**疑問点**:
- 写真なしでマークできる？
- 写真が必須の場合、写真がないとマークできない

**現在の仕様**:
- 写真はオプション
- クリックだけでマークできる

**検討事項**:
- **オプションA**: 写真なしでもOK（現在の仕様）
  - メリット: 手軽、写真撮れない場所でも遊べる
  - デメリット: 証拠がない、ずるできる
  
- **オプションB**: 写真必須（Web版っぽい）
  - メリット: 思い出が残る、達成感が高い
  - デメリット: 手間、カメラ必須

**推奨**: モード選択
- 「かんたんモード」: 写真なしOK
- 「しっかりモード」: 写真必須

---

### 4. ビンゴ達成後の動作⭐⭐
**疑問点**:
- ビンゴ達成したら次はどうなる？
- そのまま続けられる？
- 自動で新しいゲームが始まる？

**現在の仕様**:
- ビンゴメッセージが表示されるだけ
- その後の動作が不明確

**不足している要素**:
```javascript
// ビンゴ達成時の処理
onBingoAchieved() {
  // 1. ポイント付与
  // 2. 履歴に保存
  // 3. 達成画面を表示（お祝いアニメーション）
  // 4. 「続ける」or「新しいゲーム」を選択
}
```

**必要なUI**:
- ビンゴ達成画面（フルスクリーンモーダル）
- お祝いアニメーション
- 獲得ポイント表示
- 「続ける」「新しいゲーム」ボタン

### 5. お題のカテゴリ/季節/地域版⭐

- あきのおさんぽ
- ふゆのおさんぽ
- 調布版
- 多摩動物公園版

**現在の仕様**:

- しかし**季節**や**地域**の概念がない

**不足している要素**:
```javascript
topicCategories = {
  spring: ['桜', 'つくし', 'タンポポ', ...],
  summer: ['セミ', 'かき氷', '花火', ...],
  autumn: ['紅葉', 'どんぐり', 'コスモス', ...],
  winter: ['雪だるま', 'マフラー', '霜柱', ...],

```

**必要なUI**:
- ゲーム開始時にカテゴリ選択
- 「春の散歩」「夏の散歩」などのテーマ
