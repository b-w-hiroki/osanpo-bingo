# ビンゴアイコン 統一規格

## 目的
素材画像のサイズ・スタイルがバラバラだと、ビンゴ表示で見た目にばらつきが出ます。
以下の規格に統一することで視認性と一貫性を確保します。

---

## ファイル仕様

| 項目 | 規格値 | 備考 |
|------|--------|------|
| **解像度** | **512×512px** | 2のべき乗、Retina対応 |
| **形式** | PNG | 透過（アルファチャンネル） |
| **ファイル名** | `icon-{名前}.png` | 英小文字・ハイフン区切り |
| **配置** | `assets/icons/` |  |

---

## デザイン仕様（新規作成時）

| 項目 | 規格 |
|------|------|
| **トーン** | フラット、パステル、かわいい |
| **線** | クリアなアウトライン |
| **配色** | ミントグリーン・クリーム・ソフトブラウン基調 |
| **キャンバス内余白** | 図形はキャンバスの約70～80%に収める |

---

## 現状のサイズ問題（2026年2月時点）

| サイズ | アイコン数 | ファイル例 |
|--------|-----------|------------|
| 1024×1024 | 11個 | bench, happa, hashi, jitensha, kumo... |
| 2048×2048 | 10個 | hana, inu, kawa, ki, neko, tori... |

→ **512×512** に統一リサイズすることで軽量化しつつ品質を維持。

---

## リサイズ方法

### 方法1: Node.js スクリプト（推奨）

```bash
cd "（プロジェクトルート）"
npm install
node tools/resize-icons.js
```

元ファイルは `assets/icons/backup/` に退避されます。

### 方法2: 画像編集ソフトで手動

1. 各アイコンを 512×512px にリサイズ
2. 縦横比を維持しつつ中央でトリミング（またはアスペクト比1:1でリサイズ）
3. PNGで上書き保存

---

## 表示サイズ（CSS）

ビンゴセル内での表示は CSS で制御：
- PC: 4.2rem
- スマホ(480px以下): 3.2rem
- 小さいスマホ(360px以下): 2.8rem
